 ## PHASE 1 — ENDPOINT SECURITY ONBOARDING

Microsoft Defender for Endpoint (MDE)



## Phase 1 Objective

Onboard an endpoint into Microsoft Defender for Endpoint and validate security telemetry reporting.



##  Tools & Technologies
	•	Microsoft Defender for Endpoint
	•	macOS
	•	Terminal (zsh)
	•	Local onboarding script
	•	EICAR test file
	•	Microsoft Defender Portal



## Phase 1 Tasks Completed

Endpoint Onboarding

	•	Selected macOS onboarding method using local script deployment
	•	Downloaded Defender onboarding and installation packages
	•	Executed onboarding script with elevated privileges
	•	Installed Microsoft Defender Sense client

Connectivity Validation

	•	Verified Defender engine and Sense client versions
	•	Confirmed endpoint heartbeat and cloud connectivity
	•	Validated device registration in Defender portal

Security Policy Application

	•	Applied default Defender security baseline
	•	Enabled endpoint protection features
	•	Confirmed configuration enforcement from cloud portal

Detection Validation

	•	Executed EICAR test file to simulate malware detection
	•	Confirmed local Defender response
	•	Verified device reporting and telemetry ingestion

Portal Verification

	•	Validated device visibility under:
	•	Assets → Devices
	•	Endpoint inventory
	•	Reviewed device health, risk level, and configuration status



## Phase 1 Outcome
	•	Endpoint successfully onboarded into Microsoft Defender for Endpoint
	•	Security telemetry flowing correctly
	•	Device actively protected and managed
	•	Endpoint ready for detection, investigation, and response operations
